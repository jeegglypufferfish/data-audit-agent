# PRD

> 고객 기준은 [customer.md](./customer.md), 처리 흐름은 [architecture.md](./architecture.md), 작업 원칙은 [AGENTS.md](./AGENTS.md)를 참조한다.

## 문서 목적
이 문서는 Data Audit Agent의 제품 범위, MVP 기준, 성공 조건을 정리한다.

## 1. 제품 한 줄 정의
Data Audit Agent는 보고서 초안과 원출처를 비교해 검증 흔적을 구조화하고, 사람 승인 전 검토 품질을 높이는 검증 지원 도구다.

## 2. 문제 정의
보고서 검증은 수치, 표, 그림, 문단, 각주를 수작업으로 대조하는 방식에 의존한다. 이 방식은 시간이 오래 걸리고, 같은 값을 반복 확인하게 하며, 누락과 오독이 최종 승인 직전까지 남을 수 있다.

## 3. 목표
- 제출 직전 실제 불일치를 빠르게 찾는다.
- 각 검증 항목을 원출처와 연결해 재확인 비용을 줄인다.
- 결과를 사람이 검토하기 쉬운 구조로 정리한다.
- 자동 수정이 아니라 출처 근거 중심의 판단 흐름을 제공한다.
- 검토자와 책임자에게 넘길 수 있는 흔적을 남긴다.

## 4. 비목표
- 보고서 본문 전체 자동 작성
- 승인 없는 원문 자동 수정 또는 자동 반영
- 출처가 불명확한 값의 임의 보정
- 최종 발송 자동화
- 고객정보 연동
- 전사 권한 체계 완전 구현
- 외부 공식 데이터 소스 전반의 완전 자동 수집

## 5. MVP 범위
### 포함
- 단일 보고서 1건 업로드
- 원출처 문서 다건 업로드
- 검증 자료 폴더 지정
- 보고서 블록 분해
- 표, 본문, 그림 또는 그래프 숫자 비교
- 값, 기간, 단위 비교
- `일치 / 불일치 / 확인 필요` 판정
- 검토형 엑셀 생성
- 사람 검토 체크리스트 생성

### 샘플 기준
- 입력 예시: `report-draft.md`, `r-one-extract.md`
- 출력 예시: `expected-results.md`

### 제외
- 다중 보고서 동시 처리
- 승인 없는 자동 수정
- 출처 불명 값의 자동 보정
- 전체 워크플로우 자동 발송
- 정교한 권한 체계

## 6. 입력 정의
- 필수 입력: 검증용 폴더 경로
- 우선 지원 형식: PDF, DOCX, XLSX
- 선택 입력: 검증 범위 정보, 중요 수치, 확인 필요 기준, 사용자 메모
- MVP 기준 우선 지원 출처: 내부 보고서 자료, XLSX, PDF 참고 자료

## 7. 출력 정의
- 검토형 엑셀 파일 1개
- `expected-results.md` 수준의 검증 요약
- 전체 검증 목록
- 불일치 목록
- 확인 필요 목록
- 검토 체크리스트
- 재현 메모

## 8. 성공 기준
- 검증 블록별 판정이 남는다.
- 출처와 연결된다.
- 검토형 엑셀이 생성된다.
- 사람 승인 1회가 기록된다.
