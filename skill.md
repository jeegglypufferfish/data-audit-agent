# skill.md — 보고서 데이터 검증 workflow

## Context
Data Audit Agent는 보고서 초안과 출처 자료를 비교해 검증 흔적을 구조화하는 workflow skill이다. 보고서 자동 작성은 하지 않고, 제출 직전 최종 점검과 사람 승인을 지원한다.

## Goal
- 보고서 초안과 출처 자료를 비교한다.
- 일치 / 불일치 / 확인 필요를 남긴다.
- 출처 충돌과 차이 설명을 기록한다.
- 검토형 Excel 패키지와 재현 메모를 만든다.
- 최종 반영은 사람이 승인한 뒤에만 한다.
- 샘플 기준 입력은 `report-draft.md`와 `r-one-extract.md`, 샘플 기준 출력은 `expected-results.md`다.

## When to use
- 제출 직전 최종 점검이 필요할 때
- 보고서, 엑셀, PDF, 내부 참고자료를 한 폴더로 묶어 검증할 때
- 검토자에게 넘길 흔적과 체크리스트가 필요할 때

## Core workflow
1. 검증 폴더를 지정한다.
2. 보고서 후보와 출처 후보를 분류한다.
3. 보고서를 검증 블록으로 나눈다.
4. 중요 수치를 표시한다.
5. 출처 값, 기간, 기준, 단위를 비교한다.
6. `일치 / 불일치 / 확인 필요`를 판정한다.
7. 출처 충돌은 별도로 기록한다.
8. 검토형 Excel과 재현 메모를 생성한다.
9. 최종 승인자가 반영 여부를 판단한다.

## What to keep
- 표, 본문, 그림, 그래프 숫자 검증
- 출처 파일명과 위치 연결
- 문서 순서 기준 정렬
- 중요한 수치 우선 표시
- 사람 승인 지점 유지

## What not to do
- 보고서 본문 전체 자동 작성
- 승인 없는 자동 반영
- 출처 없는 값의 임의 보정
- `확인 필요`를 무리하게 `일치`로 바꾸기
- 외부망 실시간 검증을 기본값으로 사용하기

## Outputs
- 검토형 Excel 파일
- `expected-results.md` 수준의 결과 요약
- 전체 검증 목록
- 불일치 목록
- 확인 필요 목록
- 수정 제안 목록
- 재현 메모
- 최종 승인 체크리스트

## Definition of Done
- 검증 블록이 구조화되어 있다.
- 각 블록에 판정이 남아 있다.
- 출처와 근거 위치가 연결되어 있다.
- 검토형 Excel이 생성된다.
- 사람 검토가 필요한 항목이 분리되어 있다.
