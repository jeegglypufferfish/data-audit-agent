# skill.md — 보고서 데이터 검증 workflow

## Context
Data Audit Agent는 보고서 초안과 출처 자료를 비교해 검증 흔적을 구조화하는 workflow skill이다. 보고서 자동 작성은 하지 않고, 제출 직전 최종 점검과 사람 승인을 지원한다. 검증 대상은 보고서 본문에 적힌 모든 수치형 데이터, 단위, 표의 숫자와 단위이며 `낮음`, `높음` 같은 정성 수식어는 제외한다. `해당 수치의 출처`는 숫자 주변 문맥을 읽어 항목별로 파악한다. 원본 `xlsx`, `docx`, `pdf`도 셀·문장·표·각주·주석 주변 문맥을 함께 본다.

검증은 대화상에서 `보고서 먼저 → 원본 데이터 파일 다음 → 검증 시작` 순서로만 진행한다. 보고서나 원본 데이터 파일이 아직 없으면 검증을 시작하지 않고, 다음 입력을 요청한다.

## Goal
- 보고서 초안과 출처 자료를 비교한다.
- 일치 / 불일치 / 확인 필요를 남긴다.
- 출처 충돌과 차이 설명을 기록한다.
- 엑셀형 검증 결과와 재현 메모를 만든다.
- 최종 반영은 사람이 승인한 뒤에만 한다.

## When to use
- 제출 직전 최종 점검이 필요할 때
- 보고서, 엑셀, PDF, 내부 참고자료를 한 폴더로 묶어 검증할 때
- 검토자에게 넘길 흔적과 체크리스트가 필요할 때

## Core workflow
1. 보고서를 먼저 받는다.
2. 원본 데이터 파일을 이어서 받는다.
3. 두 입력이 모두 준비되면 검증 폴더를 지정한다.
4. 보고서 후보와 출처 후보를 분류한다.
5. 보고서를 검증 블록으로 나눈다.
6. 본문 수치와 표 숫자, 단위를 모두 추출하되 정성 수식어는 제외한다.
7. 중요 수치를 표시한다.
8. 출처 값, 단위, 기간, 기준과 문맥 출처를 비교한다.
9. `일치 / 불일치 / 확인 필요`를 판정한다.
10. 출처 충돌은 별도로 기록한다.
11. 엑셀 결과를 지정 와꾸로 생성한다.
12. 최종 승인자가 반영 여부를 판단한다.

## What to keep
- 본문 수치, 단위, 표 숫자와 단위 검증
- 정성 수식어는 제외하고 숫자 중심으로 검증
- 출처 파일명과 위치 연결
- 숫자 주변 문맥에서 읽은 출처 표기 반영
- 원본 xlsx/docx/pdf의 셀·문장·표·각주·주석 문맥 반영
- 문서 순서 기준 정렬
- 중요한 수치 우선 표시
- 보고서 수치가 숫자형 표현이면 보고서에 적힌 자릿수와 단위 기준으로 맞춰 반올림했을 때 같으면 일치 처리
- 금액이나 건수처럼 큰 수치는 `억`, `조`, `만` 같은 보고서 표시 단위로 먼저 맞춘 뒤 비교하고, 그 단위로 반올림했을 때 같으면 일치 처리
- `대략`, `약`, `정도`, `절반` 같은 근사 표현은 별도 근사 규칙으로 처리
- `검증완료(불일치)` 행은 전체 글씨를 빨간색으로 표시
- 사람 승인 지점 유지

## What not to do
- 보고서 본문 전체 자동 작성
- 승인 없는 자동 반영
- 출처 없는 값의 임의 보정
- `확인 필요`를 무리하게 `일치`로 바꾸기
- 외부망 실시간 검증을 기본값으로 사용하기
- 같은 이름의 `검증결과.xlsx`가 있으면 덮어쓰지 말고 `검증결과(1).xlsx`처럼 다음 번호로 저장하기

## Outputs
- 엑셀형 검증 결과
- 전체 검증 목록
- 불일치 목록
- 확인 필요 목록
- 수정 제안 목록
- 재현 메모
- 최종 승인 체크리스트

## Definition of Done
- 검증 블록이 구조화되어 있다.
- 각 블록에 판정이 남아 있다.
- 출처와 근거 위치가 연결되어 있다.
- 엑셀 결과가 생성된다.
- 사람 검토가 필요한 항목이 분리되어 있다.
