# Verification Template

이 템플릿은 Data Audit Agent의 기본 출력 형식이다.

## 사용 전제

검증은 `보고서`를 먼저 받은 뒤, `원본 데이터 파일`을 받은 다음에 시작한다.

## 출력 형식

검증 결과는 엑셀 파일 1개로 출력한다.
기본 파일명은 `검증결과.xlsx`이며, 같은 이름이 이미 있으면 `검증결과(1).xlsx`, `검증결과(2).xlsx`처럼 다음 번호를 붙여 새 파일로 저장한다.

### 시트 구성
- `검증결과` 시트 1개
- 1행은 헤더
- 2행부터 1항목 1행으로 적는다

### 열 구성
`항목 / 보고서 수치 / 단위 / 보고서 위치 / 보고서 수치 출처 / 검증여부 / 검증·비교 수치 / 단위 / 검증·비교 수치 위치 / 검증·비교 수치 출처`

## 판정 값

- `검증완료(일치)`
- `검증완료(불일치)`
- `확인 필요`

## 작성 원칙

- 항목은 보고서 안에서 실제로 검증할 지표명으로 적는다.
- 수치와 단위는 반드시 분리해서 적는다.
- `낮음`, `높음` 같은 정성 수식어는 검증 대상에서 제외하고 적지 않는다.
- 보고서 페이지는 가능한 한 정확한 페이지 기준으로 적는다.
- 보고서 내 출처는 숫자 주변 문맥을 읽어 항목별로 적고, 없으면 `출처 기재 안되어있음`으로 적는다.
- 검증 비교한 데이터는 원천 데이터의 실제 수치와 단위를 적는다.
- 원천 출처는 `xlsx`의 셀 주변, `docx`의 문장/표/각주, `pdf`의 본문/표/주석 문맥을 읽어 적는다.
- 원천 출처는 파일명, 보고서명, 표명, 페이지, 시트명, 셀 위치 등 확인 가능한 수준으로 적는다.
- 값이 반올림 또는 근사치인 경우 비교 기준을 메모에 남긴다.
- 보고서 수치가 숫자형 표현이면, 원천 수치를 보고서에 적힌 자릿수와 단위 기준으로 맞춰 반올림했을 때 같으면 `검증완료(일치)`로 판단한다.
- 금액이나 건수처럼 큰 수치는 `억`, `조`, `만` 같은 보고서 표시 단위로 먼저 맞춘 뒤 비교한다. 원천값이 보고서의 큰 단위 표현으로 반올림했을 때 같으면 `검증완료(일치)`로 판단한다.
- `대략`, `약`, `정도`, `절반` 같은 근사 표현은 별도 근사 규칙으로 판단한다.
- `검증여부`가 `검증완료(불일치)`이면 해당 행 전체 글씨를 빨간색으로 표시한다.
