# screen_spec.md — Data Audit AI Agent 화면 기획서

## 0. 문서 목적
이 문서는 `Data Audit AI Agent`의 단일 검증 화면을 기획하기 위한 상세 명세다.  
범위는 화면 구조, 사용자 흐름, 상태 정의, 컴포넌트, UX 라이팅, 보안 안내, 접근성까지만 다룬다.  
검색 로직, 비교 로직, AI 판정 로직의 구현은 포함하지 않는다.

---

## 1. 화면 목적
사용자가 `검증 대상 보고서`와 `Source File`을 업로드하면, AI Agent가 보고서의 수치형 데이터를 추출해 Source File과 비교하고, 결과를 즉시 확인하고 내려받을 수 있게 한다.

### 핵심 목적
- 검증 대상과 Source File을 빠르게 업로드한다.
- 검증 가능 여부를 먼저 확인한다.
- 일치 / 불일치 / 누락 / 검증 불가를 한 화면에서 구분한다.
- 담당자 확인이 필요한 항목을 우선적으로 드러낸다.
- 검증 결과를 Excel 리포트로 내려받는다.

### 화면 성공 기준
- 사용자가 파일 업로드에서 결과 확인까지 막힘 없이 이어진다.
- 검증 결과를 요약과 상세로 나누어 빠르게 읽을 수 있다.
- 불일치와 검증 불가 항목이 숨겨지지 않는다.
- 보안 경고와 범위 안내가 업로드 전에 충분히 보인다.

---

## 2. 사용자 목표
### 주요 사용자
- 보고서 작성자
- 검토자
- 담당자 / 승인자

### 사용자 목표
- 보고서 수치가 원본과 맞는지 빠르게 확인하고 싶다.
- 문제 항목만 먼저 보고 싶다.
- 어떤 값이 왜 불일치인지 근거를 보고 싶다.
- 검증 결과를 파일로 저장해 공유하고 싶다.
- 개인정보가 포함될 수 있는 파일을 안전하게 다루고 싶다.

## 3. 정보구조 IA

### 3-1. 화면 우선순위
1. 파일 업로드와 검증 시작
2. 검증 상태와 진행 피드백
3. 결과 요약 카드
4. 불일치 / 누락 / 검증 불가 상세
5. 근거 영역
6. 다운로드 영역
7. 시스템 안내 및 보안 고지

### 3-2. 상단에 배치할 것
- 화면 제목과 한 줄 설명
- 업로드 영역 2개
  - 검증 대상 보고서
  - Source File
- 검증 시작 버튼
- 파일 형식 안내
- 개인정보 경고 배너

### 3-3. Accordion 처리할 것
접기 가능한 영역은 결과 읽기 부담을 줄이기 위해 아래와 같이 구성한다.
- 상세 결과 테이블
- 불일치 결과 카드 묶음
- 누락 / 검증 불가 항목 묶음
- 근거 영역
- 다운로드 영역
- 시스템 안내

### 3-4. 결과 요약과 상세 결과 우선순위
- 결과 요약 카드가 항상 상세 결과보다 먼저 보인다.
- 요약은 `총 검증 항목 수 / 일치 수 / 불일치 수 / 누락 수 / 검증 불가 수` 순으로 보여준다.
- 상세 결과는 기본적으로 접힌 상태로 두고, 요약에서 문제 항목만 바로 펼칠 수 있게 한다.
- 불일치와 검증 불가 항목은 자동으로 상단 고정 또는 우선 노출한다.

### 3-5. 모바일 우선 구조
- 모바일에서는 1열 세로 구조로 배치한다.
- 업로드 영역과 검증 시작 버튼을 최상단에 둔다.
- 결과 요약 카드는 가로 스와이프 가능한 카드 스트립 또는 세로 누적형으로 제공한다.
- 상세 테이블은 카드형 리스트로 변환하거나, 행을 탭해 펼치는 구조로 단순화한다.
- 근거 영역과 다운로드 영역은 하단 아코디언으로 둔다.

---

## 4. 핵심 사용자 흐름
1. 사용자가 검증 대상 보고서를 업로드한다.
2. 사용자가 Source File을 업로드한다.
3. 시스템이 파일 형식과 읽기 가능 여부를 검증한다.
4. 시스템이 업로드 상태를 안내한다.
5. 사용자가 `검증 시작`을 누른다.
6. AI Agent가 보고서의 수치형 데이터를 추출하고 Source File과 비교한다.
7. 결과가 `일치 / 불일치 / 누락 / 검증 불가`로 정리된다.
8. 요약 카드가 먼저 표시된다.
9. 사용자는 상세 결과와 근거를 확인한다.
10. 사용자는 Excel 검증 리포트를 다운로드한다.
11. 담당자 확인 필요 항목을 재검토한다.

## 5. 화면 상태 6종

| 상태 | 화면 구성 | 주요 UX |
|---|---|---|
| 초기 상태 | 업로드 영역, 안내 배너, 빈 결과 영역, 비활성 검증 시작 버튼 | 무엇을 먼저 해야 하는지 명확히 보여준다 |
| 파일 업로드 중 | 파일명, 진행 바, 용량/형식 체크, 취소 또는 교체 버튼 | 업로드 성공 여부를 즉시 알린다 |
| 업로드 완료 | 두 파일 카드가 채워진 상태, 유효성 통과 메시지, 검증 시작 버튼 활성화 | 다음 행동을 바로 유도한다 |
| AI Agent 검증 진행 중 | 로딩 스피너, 단계 진행 표시, 현재 처리 중 문구, 잠금 상태의 결과 영역 | 기다림을 불안하지 않게 만든다 |
| 검증결과 추출 | 요약 카드, 상세 결과, 불일치 카드, 근거 영역이 채워진 상태 | 결과를 빠르게 읽고 우선순위를 잡게 한다 |
| 검증결과 엑셀 다운로드 | 다운로드 완료 토스트, 파일명, 재다운로드 버튼, PDF 다운로드 보조 액션 | 저장 완료와 다음 행동을 분명히 알려준다 |

### 상태별 추가 UX
- 초기 상태: 빈 화면이 아니라 예시 구조와 안내 문구를 보여준다.
- 업로드 중: 한 파일만 올렸을 때도 나머지 파일이 필요하다는 점을 부드럽게 안내한다.
- 업로드 완료: 파일 제거/교체가 가능해야 한다.
- 검증 중: 버튼은 비활성화하되 진행 취소 또는 파일 교체는 제한적으로 허용할 수 있다.
- 결과 상태: 불일치와 검증 불가가 먼저 눈에 들어오도록 시각적 강조를 둔다.
- 다운로드 상태: 사용자가 파일 저장 여부를 확인할 수 있도록 `다운로드 완료` 피드백을 즉시 보여준다.

## 6. 주요 컴포넌트 명세

### 6-1. 파일 업로드 영역
#### 구성
- 검증 대상 보고서 업로드
- Source File 업로드
- Drag & Drop 지원
- 파일 형식/용량/읽기 가능 여부 안내

#### 동작
- PDF, Word, Excel 등 업로드 가능
- 파일 선택 버튼과 드래그 앤 드롭을 동일 우선순위로 제공
- 업로드 후 파일명, 형식, 크기를 보여준다
- 교체 / 삭제 버튼 제공

#### 상태 표시
- 대기
- 업로드 중
- 업로드 완료
- 형식 오류
- 읽기 실패

### 6-2. 검증 실행 영역
#### 구성
- `검증 시작` 버튼
- 파일 유효성 안내
- 검증 범위 안내

#### 동작
- 두 파일이 모두 준비되어야 활성화한다
- 실행 전에는 `검증 시작`이 주 버튼이다
- 실행 중에는 상태를 `검증 진행 중`으로 전환한다

### 6-3. 결과 요약 카드
#### 구성
- 총 검증 항목 수
- 일치 수
- 불일치 수
- 누락 수
- 검증 불가 수

#### 표현 방식
- 숫자는 크게, 라벨은 작게 둔다
- 불일치와 검증 불가는 색과 배지로 강조한다
- 모바일에서는 카드 1행 2열 또는 가로 스크롤 카드로 구성한다

### 6-4. 상세 결과 테이블
#### 컬럼
- 항목명
- 보고서 값
- Source 값
- 결과
- 비고

#### 동작
- 기본은 테이블
- 모바일에서는 카드형 리스트 또는 행 펼침형으로 전환
- 결과 컬럼은 색상과 배지로 구분
- 비고에는 차이값, 단위 불일치, 출처 불명, 담당자 확인 필요를 넣는다

### 6-5. 불일치 결과 카드
#### 구성
- 항목명
- 차이값
- 보고서 값 / Source 값
- 담당자 확인 필요 배지

#### 동작
- 불일치 항목만 모아서 별도 카드로 보여준다
- 차이값이 큰 항목을 상단에 우선 정렬한다
- 카드 클릭 시 근거 영역으로 이동한다

### 6-6. 근거 영역
#### 구성
- 보고서 참조 위치
- Source 참조 위치
- AI Agent 비교 근거

#### 동작
- 선택한 행의 근거를 보여준다
- 기본은 접힘 상태
- 필요 시 한 항목씩 펼쳐본다

### 6-7. 다운로드 영역
#### 구성
- Excel 다운로드
- PDF 리포트 다운로드

#### 동작
- Excel을 기본 우선 다운로드로 둔다
- PDF는 보조 형식으로 제공한다
- 다운로드 완료 후 재다운로드 버튼을 유지한다

### 6-8. 시스템 안내
#### 구성
- 면책 배너
- 검증 범위 안내

#### 동작
- 상시 노출 또는 업로드 직전 노출
- 사용자가 범위 밖을 기대하지 않도록 명시

## 7. UX 라이팅

### 7-1. 화면 타이틀
- `Data Audit AI Agent`
- 보조 문구: `보고서 수치와 Source File을 비교해 검증 결과를 한눈에 확인합니다.`

### 7-2. 업로드 버튼
- `검증 대상 보고서 업로드`
- `Source File 업로드`
- 보조 문구: `PDF, Word, Excel 파일을 드래그 앤 드롭하거나 클릭해 선택하세요.`

### 7-3. 검증 시작 버튼
- `검증 시작`
- 비활성 상태: `파일 2개를 모두 업로드하면 검증을 시작할 수 있습니다.`

### 7-4. 플레이스홀더 / 빈 상태 문구
- `아직 업로드된 파일이 없습니다. 검증 대상 보고서와 Source File을 추가해 주세요.`
- `검증 결과가 여기에 표시됩니다.`

### 7-5. 진행 상태 문구
- `파일을 확인하는 중입니다.`
- `보고서의 수치형 데이터를 추출하고 있습니다.`
- `Source File과 비교 검증을 진행하고 있습니다.`
- `결과를 정리하는 중입니다.`

### 7-6. 완료 상태 문구
- `검증이 완료되었습니다.`
- `Excel 리포트를 다운로드할 수 있습니다.`
- `담당자 확인이 필요한 항목을 먼저 확인해 주세요.`

### 7-7. 오류 문구
- 파일 형식 오류: `지원하지 않는 형식입니다. PDF, Word, Excel 파일만 업로드해 주세요.`
- 읽기 실패: `파일을 읽을 수 없습니다. 다른 형식으로 다시 업로드해 주세요.`
- 분석 실패: `검증 중 문제가 발생했습니다. 파일을 다시 확인한 뒤 재시도해 주세요.`
- 범위 밖 질문: `이 화면은 수치 검증 전용입니다. 검증 대상 보고서와 Source File을 업로드해 주세요.`
- 검증 불가: `현재 파일만으로는 비교가 어려워 검증 불가로 분류했습니다.`

### 7-8. 배지 / 라벨
- `일치`
- `불일치`
- `누락`
- `검증 불가`
- `담당자 확인 필요`

## 8. 개인정보 및 보안 UX

### 8-1. 업로드 전 경고
업로드 버튼 클릭 전 또는 파일 드롭 직후에 다음 경고를 보여준다.
- `개인정보가 포함된 파일인지 먼저 확인해 주세요.`
- `주민등록번호, 계좌번호, 카드번호, 휴대전화번호, 고객 식별번호가 포함되면 마스킹이 필요할 수 있습니다.`

### 8-2. 마스킹 정책
- 화면 미리보기나 결과 요약에서는 민감 정보의 일부를 마스킹한다.
- 원본 파일 자체는 임의 수정하지 않는다.
- 다운로드 결과물에도 민감 정보가 노출되지 않도록 요약 수준으로 제한한다.
- 민감 정보가 의심되면 `개인정보 포함 가능` 배지를 붙인다.

### 8-3. 보안 안내 톤
- 경고는 과도하게 위협적이지 않게, 그러나 명확하게 전달한다.
- 예시: `업로드한 파일에 개인정보가 포함되어 있을 수 있습니다. 민감 정보는 마스킹하거나 삭제한 뒤 사용해 주세요.`

## 9. 접근성 가이드

### 9-1. 웹 우선
- 데스크톱과 모바일 브라우저에서 동일 기능을 제공한다.
- 마우스 없이도 전 기능을 사용할 수 있어야 한다.

### 9-2. 색 대비
- 본문 텍스트와 배경은 WCAG AA 수준 대비를 만족한다.
- 상태 색상만으로 의미를 전달하지 않고 라벨도 함께 제공한다.

### 9-3. 키보드 접근성
- 탭 순서: 업로드 영역 → 파일 카드 → 검증 시작 버튼 → 결과 요약 → 상세 결과 → 근거 영역 → 다운로드 영역
- `Enter`와 `Space`로 버튼과 펼침 동작을 수행할 수 있어야 한다.
- 파일 선택 대화상자도 키보드로 접근 가능해야 한다.

### 9-4. 스크린리더 읽기 순서
- 화면 제목
- 안내 배너
- 업로드 영역
- 검증 시작 버튼
- 상태 메시지
- 결과 요약
- 상세 결과
- 근거
- 다운로드

### 9-5. 포커스 이동 규칙
- 업로드 완료 후 다음 가능한 요소로 포커스를 이동시킨다.
- 검증 완료 후 결과 요약의 첫 카드 또는 첫 불일치 항목으로 포커스를 이동시킨다.
- 오류 발생 시 오류 메시지로 포커스를 이동시킨다.

### 9-6. 상태 변화 알림
- 업로드 성공, 검증 시작, 검증 완료, 다운로드 완료는 `aria-live`로 알린다.
- 진행 중 상태는 반복적으로 과도하게 읽히지 않게 한다.

### 9-7. 파일 업로드 접근성
- 드래그 앤 드롭 외에 항상 명시적 버튼을 제공한다.
- 파일명, 용량, 형식을 텍스트로 읽을 수 있게 제공한다.
- 읽기 실패 시 대체 행동을 제시한다.

## 10. 화면 레이아웃 요약

### 데스크톱
- 상단: 타이틀 + 안내 배너 + 업로드 영역
- 중단: 검증 시작 버튼 + 상태
- 하단: 결과 요약 카드 + 상세 테이블 + 불일치 카드 + 근거 + 다운로드

### 모바일
- 상단: 타이틀 + 업로드 영역
- 중단: 검증 시작 버튼 + 상태
- 하단: 요약 카드 → 문제 항목 → 상세 결과 → 근거 → 다운로드
- 상세 정보는 접기 구조를 기본으로 한다
