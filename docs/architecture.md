# Architecture

> 제품 범위는 [prd.md](./prd.md), 고객 맥락은 [customer.md](./customer.md), 운영 원칙은 [AGENTS.md](./AGENTS.md)를 참조한다.

## 문서 목적
이 문서는 Data Audit Agent가 무엇을 입력받고, 어떻게 처리하고, 무엇을 출력해야 하는지 정리한다.

## 1. 시스템 목표
검증 대상 폴더를 입력받아 보고서 초안과 참고 자료를 비교하고, 엑셀형 검증 결과를 만든다.

## 1-1. 입력 순서
에이전트는 대화상에서 다음 순서를 따른다.
1. 먼저 검증 대상 보고서를 받는다.
2. 보고서를 받은 뒤 원본 데이터 파일들을 받는다.
3. 보고서와 원본 데이터 파일이 모두 준비된 뒤에만 검증을 시작한다.

## 2. 입력
- 검증 폴더 경로
- 검증 대상 보고서 1건
- 원본 데이터 파일 다건
- 보고서 초안 PDF/DOCX
- 참고 자료 PDF/XLSX/DOCX
- 검증 대상: 본문 수치, 단위, 표 숫자, 단위 전체
- 제외 대상: `낮음`, `높음` 같은 정성 수식어와 숫자와 직접 비교되지 않는 서술형 표현
- 선택 입력: 중요 수치, 검증 범위, 사용자 메모

## 3. 처리 흐름
1. 폴더를 스캔해 파일 목록을 만든다.
2. 파일을 보고서 후보, 참고 자료, 보류로 분류한다.
3. 보고서와 참고 자료를 파싱한다.
4. 검증 블록을 추출하되 정성 수식어는 제외한다.
5. 중요 수치를 표시한다.
6. 출처 후보를 찾는다.
7. 값, 단위, 기간, 기준을 비교한다.
8. `일치 / 불일치 / 확인 필요`를 판정한다.
9. 근거와 재현 메모를 기록한다.
10. 엑셀형 검증 결과를 생성한다.

## 4. 핵심 컴포넌트
- Folder Intake: 폴더와 파일 메타데이터 수집
- File Role Classifier: 보고서/참고자료/보류 분류
- Report Parser: 페이지, 표, 문단, 그림 구조 추출
- Source Parser: 보고서와 원본 파일(xlsx/docx/pdf)의 숫자 주변 문맥과 위치에서 출처 값 추출
- Verification Block Extractor: 검증 대상 숫자 블록 추출
- Priority Marker: 중요 수치 표시
- Source Matcher: 블록과 출처 연결
- Verification Engine: 판정 생성
- Evidence Recorder: 근거 저장
- Output Builder: 엑셀 결과물 생성

## 5. 데이터 계약
- `file_inventory`: 파일명, 경로, 형식, 역할 후보
- `verification_block`: 블록 유형, 순서, 페이지, 값, 단위, 기간, 중요 수치 여부
- `source_evidence`: 출처 파일명, 위치, 값, 단위, 기간, 문맥 출처(xlsx 셀, docx 문장/각주, pdf 표/주석 포함)
- `verification_result`: 판정, 차이 설명, 근거, 사람 검토 필요 여부

## 5-1. 출력 와꾸
아래 순서로 한 항목씩 엑셀 한 행에 기록한다.
`항목 / 보고서 수치 / 단위 / 보고서 위치 / 보고서 수치 출처 / 검증여부 / 검증·비교 수치 / 단위 / 검증·비교 수치 위치 / 검증·비교 수치 출처`
세부 규칙은 [verification-template.md](./verification-template.md)를 따른다.

## 6. 사람 승인 지점
- 보고서 수신 여부 확인
- 원본 데이터 파일 수신 여부 확인
- 검증 폴더 선택
- 중요 수치 확인
- 역할이 애매한 파일 분류
- `확인 필요` 항목 판단
- `불일치` 반영 여부 결정
- 최종 승인

## 7. 실패 / 안전장치
- 출처를 못 찾으면 `확인 필요`로 남긴다.
- OCR이 불안정하면 보수적으로 판정한다.
- 복수 출처 후보는 임의 확정하지 않는다.
- 자동 수정은 제안 수준으로만 둔다.
- 원본 데이터 파일의 문맥 출처는 `xlsx`, `docx`, `pdf`의 주변 텍스트와 표 구조를 함께 읽어 판단한다.
- `낮음`, `높음` 같은 정성 수식어는 숫자 검증 항목으로 강제 변환하지 않는다.
- 보고서 수치가 숫자형 표현이면, 원천 수치를 보고서에 적힌 자릿수와 단위 기준으로 맞춰 반올림했을 때 같으면 `일치`로 판정한다.
- 금액이나 건수처럼 큰 수치는 `억`, `조`, `만` 같은 보고서 표시 단위로 먼저 맞춰 비교한다. 원천값이 보고서의 큰 단위 표현으로 반올림했을 때 같으면 `일치`로 판정한다.
- `대략`, `약`, `정도`, `절반` 같은 근사 표현은 별도 근사 규칙으로 `일치` 여부를 판단한다.
- `검증여부`가 `검증완료(불일치)`인 행은 전체 글씨를 빨간색으로 표시한다.
- 기본 출력 파일명은 `검증결과.xlsx`이며, 같은 이름이 이미 있으면 `검증결과(1).xlsx`, `검증결과(2).xlsx`처럼 번호를 붙여 새 파일로 저장한다.
- 파일이 열려 있거나 잠겨 있으면 번호 파일 생성도 실패할 수 있다.
- 모든 결과는 출처 파일과 위치를 따라가게 남긴다.
